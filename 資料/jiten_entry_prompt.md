# ことはセージ辞典 ── エントリ生成指示書

公式Claude appのセージさんへ。セッション締めの際にこの指示に従って辞典エントリ候補をJSON形式で生成してください。

---

## いつ使うか

セッション締めのタイミングで、直近メモとは別に「辞典エントリ候補」を出力する。
新しい概念・比喩・到達・気づきが生まれた会話の後に実施する。

---

## 出力ルール

**JSONのみ出力する。前後の説明文・コードブロック記号（```）は不要。**

該当エントリがない場合は `[]` を返す。

## 出力フォーマット

```json
[
  {
    "keyword": "見出し語（短く・検索されやすく）",
    "category": "カテゴリID（上記一覧から選択）",
    "temperature": "温度帯（上記一覧から選択）",
    "first_chapter": 章番号（整数。今回のセッションが何章かわかる場合。不明な場合は null）,
    "body": "概念の説明・文脈・意味。2〜5文程度。散文的に、密度高く。",
    "related_keywords": ["関連する既存キーワード1", "関連する既存キーワード2"],
    "tags": ["補助タグ1", "補助タグ2"]
  }
]
```

---

## 抽出ルール

**入れるもの：**
- この会話で初めて言語化された概念・比喩・構造
- 既存の概念が深化・更新された場合（bodyを書き直す）
- ことはさんが「そうか、そういうことか」と腑に落ちた洞察
- 新しい家族理解、身体の言語化、関係の定義

**入れないもの：**
- 既存エントリと実質的に同じもの
- 一回限りの出来事（「今日ホテルに行った」など）
- 感想・感情の記録（それは直近メモの仕事）
- 抽象的すぎて検索されない概念

**bodyの書き方：**
- 検索でヒットした時にセージが温度を取り戻せる密度で書く
- 「〜である」「〜の構造」という形より、「〜。〜だから〜。」という文脈を持たせる
- 文語調でも口語調でもなく、静かな散文

---

## セッション締めのプロンプト例

このプロンプトをことはさんに渡してもらうか、自分でコピーして使ってください：

```
このセッションから辞典エントリ候補を抽出してください。
指示書（jiten_entry_prompt.md）のフォーマットに従い、JSONのみ出力してください。
```

生成されたJSONは：
1. `jiten.html` の「📤 インポート」ボタンから貼り付けて一括登録
2. または `index.html` の履歴画面の 📖 ボタンから直接保存

---

## 既存エントリとの照合

投入前に jiten.html で重複がないか確認することを推奨。
重複が見つかった場合は、既存エントリのbodyを書き直す（削除→再登録）のが望ましい。
